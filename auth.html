PARADIGMAS DE LINGUAGENS DE PROGRAMAÇÃO

PARADIGMA IMPERATIVO

Definição:
O paradigma imperativo descreve a computação como uma sequência de comandos que alteram o estado do programa. É baseado em "como" fazer algo, utilizando instruções passo a passo (variáveis, loops, condicionais).


Aplicações na Indústria:
- Sistemas embarcados (controle de hardware).
- Aplicações de baixo nível (drivers, kernels de SO).
- Scripting (Bash, Python para automação). 



                            // Exemplo: Função que calcula a soma (JAVASCRIPT)
                            function sum(a, b) {
                                return a + b;
                            }
                            console.log(sum(2, 3));
Output: 5

PARADIGMA DECLARATIVO

Definição:
O paradigma declarativo descreve o que o programa deve realizar, sem especificar como alcançá-lo, focando na lógica do resultado.


Aplicações na Indústria:
- Consultas em bancos de dados (SQL).
- Processamento de dados (Pandas, Spark).
- Desenvolvimento de interfaces (HTML, CSS). 



                            SELECT nome FROM usuarios WHERE idade > 18;
Output: [Lista de nomes]

PARADIGMA LÓGICO

Definição:
O paradigma lógico baseia-se em lógica formal, onde o programa é uma coleção de fatos e regras que o sistema usa para inferir resultados.

Aplicações na Indústria:
- Sistemas especialistas.
- Inteligência artificial (Prolog).
- Raciocínio automático. 



                            % Exemplo: Fato e regra em Prolog
                            pai(joao, maria).
                            ancestral(X, Y) :- pai(X, Y).
                            ancestral(X, Y) :- pai(X, Z), ancestral(Z, Y).
Output: true (se joao for ancestral de maria)

PARADIGMA FUNCIONAL

Definição:
O paradigma funcional trata a computação como a avaliação de funções matemáticas, evitando estados e mutações de dados.


Aplicações na Indústria:
- Processamento de dados (Haskell, Scala).
- Desenvolvimento de APIs (Elixir).
- Análise de big data. 



                            -- Exemplo: Função que dobra um valor (Haskell)
                            dobro x = x * 2
                            main = print (dobro 5)
Output: 10

PARADIGMA ORIENTADO A EVENTOS

Definição:
O paradigma orientado a eventos (POE) baseia-se em eventos que disparam a execução de código, comum em interfaces gráficas e sistemas reativos.


Aplicações na Indústria:
- Interfaces gráficas (JavaScript, Qt).
- Jogos (Unity).
- Aplicações em tempo real. 



                            // Exemplo: Evento de clique (JavaScript)
                            document.addEventListener('click', () => console.log('Clicou!'));
Output: Clicou! (ao clicar)

PARADIGMA ORIENTADO A OBJETOS

Definição:
O paradigma orientado a objetos (POO) organiza o código em objetos que combinam dados e comportamentos, usando conceitos como encapsulamento e herança.


Aplicações na Indústria:
- Desenvolvimento de software (Java, C#).
- Jogos (C++).
- Sistemas empresariais. 



                            // Exemplo: Classe simples (Java)
                            class Carro {
                                String marca = "Toyota";
                                public static void main(String[] args) {
                                    Carro meuCarro = new Carro();
                                    System.out.println(meuCarro.marca);
                                }
                            }
Output: Toyota

PARADIGMA CONCORRENTE

Definição:
O paradigma concorrente permite a execução de múltiplas tarefas simultaneamente, gerenciando a coordenação entre elas.


Aplicações na Indústria:
- Sistemas distribuídos (Go, Erlang).
- Servidores (Java, C++).
- Aplicações multithread. 



                            // Exemplo: Goroutine simples (Go)
                            package main
                            import "fmt"
                            func main() {
                                go fmt.Println("Concorrente!")
                                fmt.Println("Sequencial!")
                            }
Output: Sequencial!

MULTIPARADIGMA

Definição:
O paradigma multiparadigma combina diferentes abordagens (imperativo, funcional, etc.) para oferecer flexibilidade no desenvolvimento.


Aplicações na Indústria:
- Desenvolvimento web (Python, JavaScript).
- Sistemas complexos (C++, Ruby).
- Ferramentas de automação. 



                            # Exemplo: Mistura imperativa e funcional (Python)
                            def soma(a, b): return a + b  # Funcional
                            x = 2
                            y = 3
                            print(soma(x, y))  # Imperativo
Output: 5
